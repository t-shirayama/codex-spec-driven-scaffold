# SPEC-001 User Profile Update

## Summary

ログイン済みユーザーが、自分の表示名と自己紹介文を更新できるようにする。プロフィール情報は保存後すぐに画面へ反映され、他ユーザーが閲覧するプロフィール表示にも同じ値が使われる。

## Background

ユーザーがサービス内で自分を識別しやすくするため、最低限のプロフィール編集機能が必要である。メールアドレス、権限、アカウント状態の変更はセキュリティ影響が大きいため、このspecでは扱わない。

## Related Requirements

- `FR-001`: ユーザーは自分のプロフィールを管理できる
- `NFR-SEC-001`: ユーザーは自分以外のプロフィールを更新できない

## Scope

### In Scope

- ログイン済みユーザーが自分の表示名を更新できる。
- ログイン済みユーザーが自分の自己紹介文を更新できる。
- 更新成功後、保存済みのプロフィール情報を返す。
- 未ログインユーザー、別ユーザー更新、入力不正時のエラーを扱う。

### Out of Scope

- メールアドレス、パスワード、アイコン画像、権限の変更。
- 管理者による代理更新。
- プロフィール公開範囲の設定。

## Behavior

### Normal Behavior

- ユーザーはプロフィール設定画面で表示名と自己紹介文を編集する。
- 保存時、クライアントは `PATCH /api/me/profile` を呼び出す。
- サーバーは認証済みユーザー自身のプロフィールだけを更新する。
- 更新成功時、サーバーは保存済みの `displayName`、`bio`、`updatedAt` を返す。
- クライアントはレスポンスを画面へ反映し、成功メッセージを表示する。

### Error Behavior

- 未ログインの場合は `401` を返し、ログイン導線を表示する。
- 別ユーザーのプロフィール更新はAPIとして提供しない。サーバーは認証ユーザーIDだけを更新対象にする。
- `displayName` が空、または50文字を超える場合は `400` を返す。
- `bio` が160文字を超える場合は `400` を返す。
- 想定外の保存失敗では `500` を返し、再試行可能なエラー表示にする。

### Edge Cases

- `bio` は空文字を許可する。
- 前後の空白だけで構成された `displayName` は空として扱う。
- 同じ値で保存した場合も成功として扱い、現在の保存値を返す。
- 連続保存は後勝ちとし、レスポンスの `updatedAt` で最新状態を判断する。
