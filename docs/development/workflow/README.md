# Workflow

ブランチ運用、コミット、PR、レビューの方針を記述します。

## Flow

1. 要件またはspecを作成する。
2. `tasks.md` に実装、テスト、ドキュメント更新の作業を分解する。
3. Codexまたは開発者が実装する。
4. テストを実行し、結果を記録する。
5. レビューで仕様逸脱、テスト不足、更新漏れを確認する。

## PR Checklist

- [ ] 対象specが明示されている。
- [ ] 受け入れ条件を満たしている。
- [ ] 必要なテストが追加または更新されている。
- [ ] 関連ドキュメントが更新されている。
- [ ] 実行したコマンドが説明されている。
