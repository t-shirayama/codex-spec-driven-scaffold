# Repository Structure

実装コード側のディレクトリ構成方針を記述します。

## Rule

- 新しいファイルは、既存の責務境界に沿った場所へ置く。
- 似た目的のファイルが既にある場合は、まず既存パターンを確認する。
- spec、テスト、実装の対応関係が追えるようにする。
- 例外的な配置をした場合は、理由を `plan.md` またはADRに残す。

## Project Structure

実プロジェクトでは、ここに実装コードの主要ディレクトリと責務を書きます。

| Path | Responsibility |
|---|---|
| `src/` | 実装コードの置き場 |
| `tests/` | テストコードの置き場 |
