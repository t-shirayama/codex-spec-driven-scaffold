# Naming

ファイル名、ディレクトリ名、変数名、関数名、API名などの命名規則を記述します。

## General

- 既存の命名規則を優先する。
- 同じ概念には同じ名前を使う。
- プロダクト固有の用語は `docs/product/glossary/` に合わせる。
- 略語は一般的なもの、または `abbreviations.md` に登録したものだけ使う。

## Documents

- specフォルダは `001-feature-name` のように3桁連番と英小文字ハイフンにする。
- ADRは `0001-record-architecture-decisions.md` のように4桁連番にする。
- 画面フォルダは英小文字ハイフンにする。
