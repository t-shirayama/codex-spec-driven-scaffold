# Dependencies

依存関係の追加・更新・削除方針を記述します。

## Rule

- 新しい依存を追加する前に、標準機能や既存依存で解けないか確認する。
- 同じ目的の依存を重複して入れない。
- ライセンス、メンテナンス状況、バンドルサイズ、セキュリティ影響を確認する。
- 依存追加の理由はPRまたは `plan.md` に書く。
