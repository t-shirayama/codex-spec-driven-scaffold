# Testing

UT / IT / E2E の方針、テスト観点、品質ゲートを管理します。

ここは実際のテストコード置き場ではなく、テストの考え方、観点、完了条件を管理する場所です。機能ごとの具体的なテストケースは `docs/specs/{feature}/test-cases.md` に書きます。

## 構成

| Path | 役割 |
|---|---|
| `strategy/` | テスト戦略、品質ゲート、カバレッジ |
| `unit/` | 単体テスト方針 |
| `integration/` | 統合テスト方針 |
| `e2e/` | E2E方針 |
| `fixtures/` | テストデータ |
| `reports/` | テスト結果レポート |

## Rule

- ロジック変更には単体テストを追加または更新する。
- API、DB、外部連携などの境界変更には統合テストを追加または更新する。
- 重要なユーザーフローにはE2Eテストを追加または更新する。
- 実行できないテストがある場合は、理由と実行すべきコマンドを記録する。
