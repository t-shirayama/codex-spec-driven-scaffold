# Specs

機能単位の仕様を管理する。

## Spec folder format

```text
docs/specs/001-feature-name/
├── README.md
├── spec.md
├── plan.md
├── tasks.md
├── test-cases.md
└── api.md
```

## Spec status examples

- draft
- review
- approved
- in-progress
- done
- deprecated
