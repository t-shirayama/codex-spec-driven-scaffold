# Monitoring

アラート設計、ログ観点、SLO/SLAの定義を記述します。

## Signals

| Signal          | Purpose        | Alert Condition |
| --------------- | -------------- | --------------- |
| `<metric-name>` | 監視目的を書く | 発報条件を書く  |

## Rule

- ユーザー影響がある失敗は検知できるようにする。
- エラー率、レイテンシ、ジョブ失敗、外部連携失敗を必要に応じて監視する。
- 個人情報や秘密情報をログに出さない。
