# Operations

リリース後の運用に関する情報を管理します。

機能実装で環境、リリース、監視、障害対応に影響がある場合は、対象specと一緒に更新します。

## 構成

| Path            | 役割                                 |
| --------------- | ------------------------------------ |
| `environments/` | 環境差分、環境変数、シークレット     |
| `release/`      | リリース、デプロイ、ロールバック     |
| `runbook/`      | 障害対応、定常運用、エスカレーション |
| `monitoring/`   | ログ、メトリクス、アラート、SLO/SLA  |

## Rule

- 運用手順に影響する仕様変更は、実装と同じPRで更新する。
- 手順は、担当者が迷わず実行できる粒度で書く。
- 監視できない重要処理を追加しない。
