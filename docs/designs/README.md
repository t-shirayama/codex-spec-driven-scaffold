# Designs

画面仕様、デザイン資料、画像、UI素材を管理します。

特定のデザインツールに依存しないように、ここでは画面の目的、状態、入力、導線、アクセシビリティ観点をテキストで管理します。

## 構成

- `references/`: 外部デザイン資料、プロトタイプ、共有リンク
- `screens/`: 画面仕様
- `images/`: 画面イメージ、スクリーンショット、モック画像
- `assets/`: UI素材

## Codexに渡す情報

UI実装を依頼するときは、対象画面のREADMEと `states.md` をspecと一緒に渡します。

- 画面の目的
- 関連spec
- UI要素
- 状態ごとの表示
- 入力検証
- 遷移元と遷移先
- 画像や参考資料の場所
